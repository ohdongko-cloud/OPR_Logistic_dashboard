# OPR 물류 대시보드 — 운영 가이드

> 물류 담당자·관리자용 한국어 단계별 가이드입니다. 주 1회 RAW 업로드 → 대시보드 자동 갱신 운영을 기준으로 합니다.
> 대상 독자: ① 물류 담당자(데이터 업로드·조회) ② 관리자(권한 부여·목표/전년/비고 입력).
> 보안 원칙: 실데이터는 사내 환경에만, 외부 공유 자료는 마스킹. 시크릿(접속정보·토큰)은 화면·문서에 남기지 않습니다.

---

## 0. 한눈에 보기 (주 1회 워크플로우)

```
① SAP/BI 에서 RAW 다운로드(아이템 당월/누적 · 매장 당월)
        │
② 업로드 양식 템플릿으로 "어느 시트·어느 열" 형식 확인 (선택)
        │
③ /upload 에서 RAW(.xlsx) 업로드 → 검증 → 새 CURRENT 스냅샷 적재
        │
④ 대시보드 3뷰 자동 갱신 — 아이템·매장·상품
        │
⑤ 필요 시 PPT/엑셀 내보내기, 관리자는 목표·비고 입력
```

소요: 정상 RAW 기준 업로드~갱신 수 분 이내. 매주 같은 요일·같은 추출 조건 권장(추세 비교 일관성).

---

## 1. 로그인 (@eland.co.kr · OTP)

1. 브라우저로 대시보드 주소에 접속하면 로그인 화면(`/login`)이 뜹니다.
2. **회사 이메일(@eland.co.kr)** 을 입력하고 인증코드 요청을 누릅니다.
   - 허용 도메인 외 이메일은 로그인되지 않습니다(예외 계정은 관리자 설정).
3. 메일로 받은 **OTP(일회용 인증코드)** 를 입력하면 로그인됩니다.
   - 코드는 일정 시간 후 만료됩니다. 만료되면 다시 요청하세요.
4. 로그인 후 좌측 사이드바에서 권한이 있는 메뉴(아이템·매장·상품·업로드·입력·관리)가 보입니다.

> 권한이 없어 메뉴가 안 보이거나 "권한이 없습니다" 안내가 나오면 → 관리자에게 권한 요청(§7 참고).

## 2. RAW 다운로드 & 형식 확인

### 2-1. SAP/BI 에서 RAW 받기
- **아이템 파일**: 유통물류(OPR) 모니터링(아이템) — 당월 또는 누적. 6개 RAW 시트
  (매출상세분석·점재고·물류재고·센터입출고·기초재고(지점)·기초재고(센터)) + 칸반·물류비예측·#분류 포함.
- **매장 파일**: 유통물류(OPR) 모니터링(매장) — 당월. 5개 RAW 시트
  (매출상세분석·기말재고(지점)·기초재고(지점)·상품수불(지점)·수불오차) + 매장칸반·지점대시보드 포함.

### 2-2. 업로드 양식 템플릿으로 형식 확인 (헷갈릴 때)
RAW 의 시트·열 위치가 평소와 다른 것 같으면, `/upload` 페이지 상단의
**"업로드 양식 템플릿"** 에서 양식을 받아 대조하세요.

- **아이템 양식(6시트)** / **매장 양식(5시트)** 버튼 → 헤더가 **정확한 열 위치**에 박힌 빈 .xlsx 가 받아집니다.
- 첫 시트 **README(안내)** 에 시트별·핵심열 의미·주의가 적혀 있습니다. 예:
  - 아이템: `A열 = 계절연도-자재그룹(SKU키)`, `F열 = 구매그룹(상품)`, 매출상세 `H=실매출액·I=총매출원가·J=판매수량`.
  - 매장: `B열 = 점포코드(플랜트)`, 같은 시트에 픽스(좌)·전체(우) 블록이 가로 병렬.
- 양식에는 **실데이터가 없습니다(헤더·안내·예시 더미만)** — 형식 참고용입니다.

> 주의: SAP 추출 양식이 바뀌면 헤더 위치가 어긋나 업로드가 거부될 수 있습니다.
> 이때 템플릿과 비교해 어느 시트/열이 다른지 확인하고, 추출 조건을 맞추세요.

## 3. 업로드 (`/upload`)

> 업로드는 **입력(MANAGE) 권한**을 가진 담당자만 수행할 수 있습니다(서버에서 강제).
> 권한이 없으면 양식 다운로드는 되지만 업로드 폼은 보이지 않습니다.

1. 사이드바 → **업로드** 로 이동합니다.
2. 드롭존에 RAW(.xlsx) 를 끌어다 놓거나 클릭해 선택합니다. **아이템·매장을 동시에** 올릴 수 있습니다.
3. 옵션 확인:
   - **파일 종류**: 자동판별(권장). 자동판별이 안 되면 아이템/매장 직접 선택.
   - **기간 유형**: 자동판별(칸반 시트명의 당월/누적). 필요 시 당월/누적 직접 선택.
   - **귀속 시작일/종료일**: 스냅샷 귀속 기간(선택). 기본 종료일 = 오늘.
   - **엔진 앵커(아이템)**: 보통 "칸반 자동추출". 추출 실패 시 기본값(당월 21/30/1.22 · 누적 172/181/1.02) 또는 수동 입력.
4. **업로드** 버튼 → 검증 후 적재 결과(스냅샷 ID·갱신 대상·fact 행수·상태)가 표시됩니다.
   - 아이템 파일 1건 → **물류 핵심지표(아이템) + 상품 SCM** 두 뷰가 함께 갱신됩니다.
   - 매장 파일 1건 → **매장 SCM** 이 갱신됩니다.
5. 적재가 끝나면 하단 **스냅샷 이력** 이 새로고침됩니다.

### 업로드가 거부될 때
- "필수 RAW 시트 누락" → 6시트(아이템)/5시트(매장)·칸반이 다 들어있는지 확인.
- "매크로/외부링크 워크북 거부" → .xlsm 이나 외부 연결 포함 파일은 불가. 값만 있는 .xlsx 로 다시 추출.
- 시트·열 위치 의심 → §2-2 템플릿과 대조.

## 4. 대시보드 3뷰 보는 법

공통: 상단 **KPI 스트립**(핵심 지표 요약) → 필터/검색/내보내기 → **드릴다운 트리테이블**(접었다폈다).
트리 행의 ▶ 를 누르면 하위로 펼쳐지고, **전체펼침/전체접힘** 버튼으로 일괄 제어합니다.
열 머리글의 ↕ 를 누르면 형제 행이 그 지표로 정렬됩니다(트리 구조는 유지).

### ① 아이템 (물류 핵심지표) — `/engine`
- 계층: **전체 → 성별 → 신상/이월 → SS·FW → 시즌 → 아이템(리프)**.
- 지표: 매출·물류비·물류비율·재고일수(센터/점포)·체화비중·입출반 등.
- 아이템 리프를 클릭하면 **SKU 상세 모달**이 열립니다.
- 당월/누적 전환 가능. 목표·비고가 입력돼 있으면 우측 "목표·비고" 열에 ▲▼ 달성률·ⓘ 비고가 표시됩니다.

### ② 매장 (매장 SCM) — `/store`
- 계층: **전체 → 채널(직영/중간관리/기타) → 점포**. (당월 기준)
- 지표: 판매배수·재고일수·시즌비중·재고보유율·**(−)마이너스재고(수량·금액)**·입고/판매/재고.
- (−)재고가 음수면 빨강으로 강조됩니다(재고 건전성 경고).

### ③ 상품 (상품 SCM) — `/product`
- 계층: **전체 → 브랜드(구매그룹) → 시즌**. 블록 = 입고(매입)·상품화(물류)·판매(영업) + 비고.
- 자동 집계 필드(입고량·재고량·출고량·출고율·판매량·판매율·매총율)는 값이 채워지고,
  **원천(일자)이 없는 필드는 "—"**(▪ 마커, 가짜값 금지), **수기 필드는 ✎**(입력은 추후).

### 데이터 품질 가드 — "—" 표시의 의미
파생 비율(물류비율·재고일수·체화비중·판매율 등)에서 **분모(재고량·매출 등)가 미미**하면
14,000%·601일 같은 오해성 극단값이 나옵니다. 이를 막기 위해 해당 칸은 **흐린 "—"** 로 표기를 보류합니다.
- 셀에 마우스를 올리면 **"분모 미미 — 참고 불가"** 툴팁이 뜹니다.
- **원본 값은 그대로 보존**되며(데이터 손실 없음), 화면 표시만 보류한 것입니다.
- 분모가 충분해지면(데이터가 쌓이면) 다음 갱신에서 정상 비율로 표시됩니다.

## 5. 내보내기 (PPT / 엑셀)

### 엑셀 내보내기 (3뷰 공통)
- 각 뷰 상단의 **⬇ 엑셀** 버튼 → 현재 화면과 **동일한 컬럼**으로 드릴다운 표가 .xlsx 로 받아집니다.
- 화면에서 보류된("—") 비율은 엑셀에서도 동일 기준으로 다뤄집니다(오해성 극단값 방지).

### PPT 내보내기 (아이템 뷰)
- 아이템 뷰(`/engine`) 상단의 **⬇ PPT** 버튼 → **원본 PPT 양식 그대로** 슬라이드가 채워진 .pptx 가 받아집니다.
  - 슬1: 물류 핵심지표 / 슬2: 매장 SCM(매장 데이터 있을 때) / 슬5: 목표 대비(목표·전년·조치 입력 시).
  - 슬3·4(상품): 행↔노드 매핑 마스터가 확보되면 채워집니다(현재 공란 — 가짜값 금지).
- 당월/누적 기준을 고른 뒤 내보내면 그 기준으로 채워집니다.

> 외부로 공유할 때는 점포명·브랜드명 등 민감정보 마스킹 여부를 확인하세요.

## 6. 관리자 — 권한 부여 (`/admin`)

> 관리(admin) 권한이 있는 사용자만 접근합니다.

1. 사이드바 → **관리** 로 이동합니다.
2. 사용자 목록에서 대상 사용자의 **탭별 권한 레벨**을 설정합니다.
   - 탭: 아이템(logistics)·매장(store)·상품(product)·입력(input)·관리(admin).
   - 레벨(높을수록 권한 큼): **VIEW(조회) → INPUT(입력) → MANAGE(관리/업로드)**.
   - 예: 업로드 담당자 → `입력 = MANAGE`. 조회만 하는 사용자 → 해당 탭 `VIEW`.
3. 저장하면 즉시 반영됩니다(다음 페이지 진입부터 적용).

---

## 7. 관리자 — 목표 / 전년 / 비고 입력 (`/input`)

> 입력(input) INPUT 이상 권한이 필요합니다.

1. 사이드바 → **입력** 으로 이동합니다.
2. 노드(전체/성별/시즌/아이템 등)별로 **목표값·전년값·비고·조치**를 입력합니다.
   - 입력한 목표·전년은 아이템 뷰의 "목표·비고" 열과 PPT 슬5(목표 대비)에 반영됩니다.
3. 아이템 뷰에서 노드 행의 **✎** 아이콘으로도 해당 노드 입력으로 바로 진입할 수 있습니다(INPUT 권한 시).

> 입력값은 기간(당월/누적)에 귀속됩니다. 기간을 맞춰 입력하세요.

---

## 8. 스냅샷 이력 · 롤백 (현재 상태)

- `/upload` 하단 **스냅샷 이력** 에서 종류·기간별 **CURRENT / SUPERSEDED** 상태, 업로더, 시각, fact 행수를 봅니다(메타만 — 실수치 미표시).
- **멱등 적재**: 같은 종류·기간을 다시 업로드하면 이전 CURRENT 가 **SUPERSEDED 로 보존**되고 새 업로드가 CURRENT 가 됩니다. **이력은 삭제되지 않습니다.**
- **SUPERSEDED 복원(롤백)**: 현재 화면 버튼으로는 미제공(**후속 제공 예정**). 잘못 올린 경우, 올바른 RAW 를 다시 업로드하면 최신본이 CURRENT 가 됩니다(이전본은 이력에 남음).
- 잘못된 적재가 의심되면: ① 올바른 RAW 재업로드로 교체, ② 필요 시 관리자/개발 담당에게 이력 기준 점검 요청.

## 부록. 자주 묻는 질문

- **Q. 비율이 "—" 로 나와요.** → 그 행의 분모(재고량/매출 등)가 너무 작아 표기를 보류한 것입니다(오해성 극단값 방지). 원본 값은 보존돼 있고, 데이터가 쌓이면 정상 표시됩니다.
- **Q. 업로드했는데 대시보드가 그대로예요.** → 적재 결과가 "CURRENT" 인지, 갱신 대상(아이템/상품/매장)이 보려는 뷰와 맞는지 확인. 브라우저 새로고침.
- **Q. 매장 뷰에 누적이 없어요.** → 매장은 당월 단일 운영입니다(누적 미동봉).
- **Q. 메뉴가 안 보여요.** → 해당 탭 권한이 없습니다. 관리자에게 권한(§6) 요청.
- **Q. 외부에 공유해도 되나요?** → 실데이터는 사내만. 외부 공유물은 점포명·브랜드명 등 마스킹 후 공유.
